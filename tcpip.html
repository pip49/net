2과목 TCP/IP - 고빈출 50제 퀴즈

가장 출제 비중이 높은 핵심 과목! 정답 확인 버튼을 활용해 암기해 보세요.

📌 학습 가이드: 2과목(TCP/IP)은 IP 주소 클래스 구분, 서브넷 마스크 계산 기법, IPv4/v6 특징, 그리고 수많은 계층별 프로토콜(ARP, ICMP, 계정 프로토콜 등)의 기능 정의가 단골 출제됩니다. 반복해서 눈에 익혀두세요!
Q1. IPv4 주소는 총 몇 비트(Bit)로 구성되어 있는가?
32비트
Q2. IPv4 주소에서 하나의 옥텟(Octet)은 몇 비트를 의미하는가?
8비트 (총 4개의 옥텟으로 32비트 구성)
Q3. IPv4 주소 중 첫 번째 옥텟의 범위가 1~126인 대규모 네트워크용 클래스는?
A 클래스
Q4. IPv4 주소 중 첫 번째 옥텟의 범위가 192~223인 소규모 네트워크용 클래스는?
C 클래스
Q5. C 클래스의 기본(Default) 서브넷 마스크 주소는 무엇인가?
255.255.255.0
Q6. B 클래스의 기본(Default) 서브넷 마스크 주소는 무엇인가?
255.255.0.0
Q7. 자신의 컴퓨터 시스템이 통신망에 정상적으로 연결되었는지 테스트하기 위해 할당된 루프백(Loopback) 특수 주소는?
127.0.0.1
Q8. 호스트 식별자 비트가 모두 '0'인 주소로, 해당 네트워크 자체를 대표하는 주소를 무엇이라 하는가?
네트워크 주소
Q9. 호스트 식별자 비트가 모두 '1'인 주소로, 해당 네트워크 내 모든 컴퓨터에 데이터를 보낼 때 쓰이는 주소는?
브로드캐스트 주소
Q10. 서브넷 마스크가 255.255.255.240일 때, 하나의 서브넷당 가질 수 있는 이론상 최대 호스트 개수(주소 개수)는?
16개 (256 - 240 = 16)
Q11. IPv4의 주소 고갈 문제를 해결하기 위해 개발된 차세대 IP 주소 체계인 IPv6는 총 몇 비트인가?
128비트
Q12. IPv6 주소 체계는 몇 진수로 표기하며, 각 필드는 무엇으로 구분하는가?
16진수, 콜론(:)으로 구분 (IPv4는 10진수, 점으로 구분)
Q13. IPv6의 주요 전송 방식 3가지는 유니캐스트, 멀티캐스트, 그리고 나머지 하나는 무엇인가?
애니캐스트 (Anycast)
Q14. IPv6 전송 방식 중 송신자와 네트워크 토폴로지상 가장 가까운 단일 수신자에게 패킷을 전달하는 방식은?
애니캐스트 (Anycast)
Q15. 클래스 개념 없이 주소를 효율적으로 나누기 위해 `192.168.1.0/24` 처럼 슬래시 뒤에 네트워크 비트 수를 표기하는 방식은?
CIDR (Classless Inter-Domain Routing)
Q16. 내부 네트워크에서는 사설 IP를 쓰고, 외부 인터넷으로 나갈 때는 공인 IP 주소로 바꾸어주는 주소 변환 기술은?
NAT (Network Address Translation)
Q17. 192.168.10.0/24 네트워크에서 실제 컴퓨터(호스트)에 할당 가능한 유효한 IP 주소의 개수는 몇 개인가?
254개 (전체 256개 중 네트워크 주소와 브로드캐스트 주소 2개를 제외)
Q18. 공인 IP 주소 부족을 해결하기 위해 약속된 C 클래스 사설 IP 대역의 시작 주소는?
192.168.0.0
Q19. DHCP 서버로부터 IP를 할당받지 못했을 때 윈도우가 자동으로 할당하는 사설 주소 대역인 `169.254.X.X` 기술을 뜻하는 명칭은?
APIPA (Automatic Private IP Addressing)
Q20. 하나의 공인 IP에 여러 포트 번호를 지정하여 수많은 내부 사설 IP 호스트들이 동시에 인터넷을 쓰도록 돕는 NAT의 변형 기술은?
PAT (Port Address Translation)
/* 21~35: 계층별 프로토콜 핵심 */
Q21. 상대방의 논리적 IP 주소를 기반으로 물리적 'MAC 주소'를 알아내기 위해 사용하는 3계층 프로토콜은?
ARP (Address Resolution Protocol)
Q22. IP 주소는 없고 MAC 주소만 알고 있는 디스크리스(Diskless) 단말기가 자신의 IP 주소를 서버로부터 알아낼 때 쓰는 프로토콜은?
RARP (Reverse ARP)
Q23. 수신측이 잘 받았는지 확인하는 '3-Way Handshaking' 과정을 거치는 신뢰성 기반의 전송 계층 프로토콜은?
TCP
Q24. 헤더가 단순하여 오버헤드가 적고 데이터 전송 속도가 가장 빠른 비연결성 전송 계층 프로토콜은?
UDP
Q25. 에러 보고 및 네트워크 상태 검사 목적의 프로토콜로, ping 명령어가 사용하는 인터넷 계층의 핵심 프로토콜은?
ICMP
Q26. IP 패킷이 수명을 다해 소멸하기 전까지 거칠 수 있는 최대 라우터 수(네트워크 경유 수)를 나타내는 IP 헤더 필드는?
TTL (Time To Live)
Q27. 웹(WWW) 페이지를 주고받을 때 기본적으로 사용하며, 기본 포트 번호로 80번을 사용하는 프로토콜은?
HTTP
Q28. HTTP의 보안 취약점을 해결하기 위해 암호화(SSL/TLS) 레이어를 추가하였으며, 포트 번호 443번을 쓰는 프로토콜은?
HTTPS
Q29. 네트워크를 통해 대용량 파일을 전송하고 수신하기 위해 고안된 제어용(21번 포트) 및 데이터용(20번 포트) 결합 프로토콜은?
FTP (File Transfer Protocol)
Q30. 인터넷 상에서 이메일을 다른 서버로 '송신(보낼 때)'할 때 사용하는 응용 계층 프로토콜(포트 번호 25)은?
SMTP (Simple Mail Transfer Protocol)
Q31. 이메일 서버에 저장된 메일을 사용자의 스마트폰이나 PC로 '수신(가져올 때)'하는 프로토콜로, 메일을 가져오면 서버에서 지우는 특징을 가진 것은?
POP3 (Post Office Protocol version 3 / 110번 포트)
Q32. POP3와 달리 이메일을 로컬로 완전히 다운로드하지 않고 서버에 둔 채 여러 단말기에서 동기화해 관리할 수 있는 메일 수신 프로토콜은?
IMAP (Internet Message Access Protocol / 143번 포트)
Q33. 멀리 떨어진 원격지의 컴퓨터에 접속하여 텍스트 기반 터미널 창을 제어할 수 있게 돕지만, 암호화 기능이 없는 평문 통신 프로토콜(23번 포트)은?
Telnet (텔넷)
Q34. 네트워크 장비의 시간(시계)을 표준 시간 서버와 동기화하기 위해 사용하는 프로토콜은?
NTP (Network Time Protocol / 123번 포트)
Q35. TCP 연결 설정 과정 중 송신측이 SYN 패킷을 보내고, 수신측이 대답으로 보내는 두 번째 단계의 패킷 명칭은?
SYN-ACK 패킷
Q36. 라우터 간에 정보를 주고받을 때 사용하는 프로토콜 중, 경유하는 라우터 수(Hop Count)를 기준으로 최대 15홉 제한을 가지는 소규모용 거리벡터 라우팅 프로토콜은?
RIP (Routing Information Protocol)
Q37. 홉 카운트의 제한이 없으며, 네트워크 변화를 빠르게 반영하고 링크 상태(Link State) 알고리즘을 기반으로 최적 경로를 계산하는 대규모용 라우팅 프로토콜은?
OSPF (Open Shortest Path First)
Q38. 윈도우 명령 프롬프트(CMD) 창에서 내 컴퓨터의 IP 주소, 서브넷 마스크, 게이트웨이 정보를 한눈에 확인하는 명령어는?
ipconfig
Q39. 리눅스(Linux) 환경에서 네트워크 인터페이스 카드의 설정 상태를 확인하고 IP를 세팅할 때 쓰는 명령어는?
ifconfig
Q40. 목적지 서버까지 거쳐가는 라우터들의 경로(IP 리스트)를 추적하여 네트워크 장애 구간을 진단할 수 있게 해주는 윈도우 명령어는?
tracert (리눅스 환경에서는 traceroute)
Q41. 내 컴퓨터에 현재 연결된 네트워크 연결 상태(세션 정보), 프로토콜별 통계 및 라우팅 테이블 정보를 출력해 주는 명령어는?
netstat
Q42. 도메인 네임 서버(DNS)의 동작 상태를 점검하거나 특정 도메인의 IP 주소를 직접 쿼리(질의)해 찾아볼 때 사용하는 명령어는?
nslookup
Q43. 내 컴퓨터의 ARP 캐시 테이블에 저장된 IP 주소와 MAC 주소의 매핑 목록을 확인하는 명령어는?
arp -a
Q44. IP 헤더의 무결성을 검증하기 위해 송신 측이 계산하여 삽입하고, 수신 측이 손상 여부를 확인하는 필드는?
헤더 체크섬 (Header Checksum)
Q45. 네트워크 보안 프로토콜로, IP 계층(3계층)에서 데이터의 기밀성(암호화)과 인증을 보장하여 주로 VPN 구축에 쓰이는 기술은?
IPsec
Q46. 전송 계층(TCP/IP) 단계에서 보안을 제공하며 웹브라우저와 서버 간 보안 통신(HTTPS)의 기반이 되는 프로토콜 체계는?
SSL / TLS
Q47. 하나의 도메인에 여러 개의 IP 주소를 등록하여 사용자들이 접속할 때마다 순차적으로 IP를 분산 할당하는 DNS 기반 부하 분산 기술은?
DNS 라운드 로빈 (Round Robin)
Q48. IP 패킷을 보낼 때 송신자의 IP 주소를 가짜 주소로 속여서 타깃 시스템을 마비시키는 네트워크 공격 기법은?
IP 스푸핑 (Spoofing)
Q49. 대량의 ICMP Echo Request(ping) 패킷을 브로드캐스트 주소로 유포하여 대상 컴퓨터에 트래픽 폭탄을 떨어뜨리는 DDoS 공격 유형은?
스머프 공격 (Smurf Attack)
Q50. TCP의 연결 설정 과정(3-Way Handshake)에서 SYN 패킷만 엄청나게 보내 서버의 접속 대기 큐를 가득 채워 마비시키는 공격은?
SYN 플러딩 (SYN Flooding)
