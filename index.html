1과목. 네트워크 일반 추천 목표: 10/12 문제

인터넷과 네트워크가 도대체 어떤 규칙으로 움직이는지 큰 그림을 배우는 과목입니다.

① OSI 7계층 (데이터가 이동하는 7개의 정거장)

네트워크를 통할 때 데이터는 7개의 단계를 거칩니다. 시험에서는 "각 계층의 대표 장비나 프로콜이 뭐냐?"를 무조건 묻습니다.

  • 1계층 물리(Physical): 케이블, 리피터, 허브 (전기 신호가 오가는 진짜 물리적 선)
  • 2계층 데이터링크(Data Link): 브릿지, 스위치 / 프로토콜: MAC 주소, HDLC, PPP (옆집 컴퓨터랑 직접 통신하기)
  • 3계층 네트워크(Network): 라우터 / 프로토콜: IP, ICMP, ARP (멀리 떨어진 목적지까지 길 찾기 - 내비게이션 역할)
  • 4계층 전송(Transport): 프로토콜: TCP, UDP (데이터를 빵꾸 없이 안전하게 보낼지 결정)
  • 5~7계층 세션/표현/응용(Application): 프로토콜: HTTP(웹), FTP(파일), SMTP(메일), DNS(주소창) (사용자가 쓰는 프로그램 영역)
★ 무조건 출제되는 암기 팁
위에서 아래로 "물-데-네-전-세-표-응" 앞 글자를 따서 순서를 외우세요! 3계층 장비는 라우터, 2계층 장비는 스위치인 점이 단골 문제입니다.
② TCP vs UDP (꼼꼼한 택배 vs 묻지마 던지기)

4계층의 핵심 통신 방식 두 가지입니다.

특징 TCP (Transmission Control Protocol) UDP (User Datagram Protocol)
연결 방식 연결형 (전화걸기식, 3-Way Handshaking) 비연결형 (편지 보내기식)
신뢰성 높음 (확인 응답, 재전송 기능 있음) 낮음 (보내면 끝, 확인 안 함)
속도 비교적 느림 매우 빠름
용도 웹 브라우징(HTTP), 파일 전송(FTP) 실시간 스트리밍, 영상 통화, DNS 서비스
③ 중요 프로토콜들 이름 맞추기
  • ICMP: "네트워크야 살아있니?" 네트워크 상태를 점검하고 에러를 알려주는 프로토콜 (ping 명령어의 기반)
  • ARP: "IP 주소는 아는데, 그 컴퓨터의 진짜 주민번호(MAC 주소)가 뭐지?" 알려주는 프로토콜
  • RARP: ARP의 반대. MAC 주소를 가지고 IP 주소를 알아낼 때 사용
  • DNS: `naver.com` 같은 문자를 `223.130.195.200` 같은 숫자로 된 IP 주소로 바꿔주는 전화번호부 서비스
2과목. TCP/IP 추천 목표: 14/17 문제 (가장 중요!)

합격을 결정짓는 과목입니다. 특히 계산 문제가 출제되니 꼼꼼하게 봐두어야 합니다.

① IP 주소 체계와 서브넷 마스크 (제일 중요!)

IP 주소는 클래스(Class)라는 덩어리로 나뉩니다. 맨 앞자리 숫자만 보고 어느 클래스인지 맞출 수 있어야 합니다.

  • A 클래스: 1 ~ 126 (국가나 대기업용, 대형 네트워크)
  • B 클래스: 128 ~ 191 (중견기업, 대학용)
  • C 클래스: 192 ~ 223 (우리 집, 소규모 사무실용) *가장 많이 출제!
★ 서브넷팅(Subnetting)의 핵심 개념
IP 주소가 아까워서 하나의 큰 네트워크를 여러 개의 작은 서브넷(Subnet)으로 쪼개는 것을 말합니다. 이때 사용하는 마스크 가리개를 서브넷 마스크라고 합니다.
* 계산 팁: 기본 C클래스 서브넷 마스크는 `255.255.255.0` 입니다.
② 사설 IP 주소 (집안용 가짜 IP)

전 세계에서 단 하나뿐인 공인 IP가 부족해서, 공유기 내부에서 공짜로 돌려 쓰는 가짜 주소 공간입니다. 외부 인터넷과 직접 통신이 불가능합니다.

  • A클래스 사설: 10.0.0.0 ~ 10.255.255.255
  • B클래스 사설: 172.16.0.0 ~ 172.31.255.255
  • C클래스 사설: 192.168.0.0 ~ 192.168.255.255 (공유기 쓰면 백프로 나오는 그 주소!)
③ IPv4 vs IPv6 (주소가 고갈되어 만든 새 주소)
  • IPv4: 32비트 길이, 10진수로 표기 (예: 192.168.0.1), 점(.)으로 구분
  • IPv6: 128비트 길이, 16진수로 표기, 콜론(:)으로 구분, 애니캐스트(Anycast) 전송 방식 추가 (유니, 멀티, 애니캐스트 3종 세트)
3과목. NOS (Network Operating System) 추천 목표: 10/15 문제

Windows 서버와 Linux 서버를 다루는 방법, 계정 권한과 명령어 문제입니다.

① Windows Server 주요 개념
  • 액티브 디렉토리(Active Directory): 회사 내 모든 컴퓨터, 사용자 계정, 프린터를 한 곳에서 중앙 관리하는 윈도우 서버 시스템
  • IIS (Internet Information Services): 윈도우 서버에서 웹 서버(홈페이지 띄우기)나 FTP 서버를 운영하기 위해 사용하는 프로그램
  • RAID (하드디스크 묶기 기술): 여러 하드가 하나처럼 움직여서 고장나도 복구되게 만드는 기술
    • RAID 0 (스트라이핑): 데이터를 쪼개서 분산 저장. 속도는 제일 빠르나 하드 하나 고장 나면 다 깨짐.
    • RAID 1 (미러링): 똑같이 복사해서 두 군데 저장. 용량은 절반 손해 보지만 안정성 최고.
    • RAID 5: 최소 3개 하드 필요. 에러 체크용 데이터(패리티)를 나누어 저장하여 속도와 안정성 절충. 단골 출제!
② 리눅스(Linux) 필수 명령어

시험에 매번 나오는 리눅스 명령어 5가지입니다.

  • chmod: 파일이나 폴더의 읽기/쓰기/실행 권한(Permission)을 변경할 때 사용
  • chown: 파일의 소유자나 소유 그룹을 변경할 때 사용
  • pwd: 내가 지금 어떤 디렉토리(폴더) 안에 들어가 있는지 현재 경로 표시
  • ls: 현재 폴더 안에 무슨 파일들이 있는지 목록 보여주기
  • top: 시스템의 CPU, 메모리 사용량 등 실시간 모니터링
4과목. 네트워크 운용기기 추천 목표: 4/6 문제

눈에 보이는 기계 장비들의 특징을 맞추는 가장 간단한 과목입니다.

① 네트워크 장비 삼총사
  • 허브(Hub): 들어온 신호를 연결된 모든 컴퓨터에 멍청하게 다 뿌리는 장비 (대역폭이 분할되어 느려짐)
  • 스위치(Switch): 목적지 MAC 주소를 확인하고 원하는 컴퓨터에게만 정확하게 데이터를 쏘아주는 똑똑한 장비 (2계층)
  • 라우터(Router): 서로 다른 네트워크(예: 우리 집 공항과 네이버 공항)를 연결해 주고, 목적지까지 최적의 경로를 찾아 안내해 주는 내비게이션 장비 (3계층)
② 게이트웨이(Gateway)와 더미 허브
  • 게이트웨이: 프로토콜 구조가 전혀 다른 네트워크 세계로 나가는 '출입구' 역할 (우리 집 공유기 IP가 보통 게이트웨이 역할을 함)
  • 리피터: 선이 너무 길면 전기가 감쇄되니까, 약해진 신호를 다시 크게 증폭시켜 주는 장비
