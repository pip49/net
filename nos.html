3과목 NOS(네트워크 운영체제) - 고빈출 50제 퀴즈

Windows Server 기능 및 Linux 필수 명령어를 완벽하게 마스터하세요.

📌 학습 가이드: 3과목(NOS)은 18문항이 출제되는 고배점 과목입니다. Windows Server의 도메인/디렉토리 구조와 RAID 연산 기법, 그리고 Linux의 핵심 관리 명령어 및 권한 표기법(8진수)에 대한 개념을 꼼꼼히 정리해 두어야 고득점이 가능합니다.
Q1. Windows Server에서 회사 내 모든 사용자 계정, 컴퓨터 장비, 프린터 등의 자원을 하나의 데이터베이스로 통합 관리하는 디렉토리 서비스의 명칭은?
액티브 디렉토리 (Active Directory)
Q2. Windows Server에서 웹 서버(HTTP), FTP 서버 등을 손쉽게 구축하고 구성할 수 있도록 지원하는 인터넷 통합 서비스 부가 기능은?
IIS (Internet Information Services)
Q3. 여러 개의 하드디스크를 하나로 묶어 성능을 높이거나 안정성을 강화하는 저장장치 이중화 기술의 명칭은?
RAID (Redundant Array of Independent Disks)
Q4. RAID 레벨 중, 데이터를 여러 하드에 조각내어 분산 저장(스트라이핑)하여 속도는 가장 빠르지만 복구 기능이 전혀 없는 레벨은?
RAID 0
Q5. RAID 레벨 중, 두 개 이상의 하드디스크에 동일한 데이터를 완전 복사하여 백업(미러링)하는 방식으로 공간 효율은 50%로 낮지만 안정성이 최고인 레벨은?
RAID 1
Q6. 최소 3개의 하드디스크가 필요하며, 에러 체크용 데이터인 패리티(Parity) 정보를 각 하드에 분산 저장하여 디스크 하나가 고장 나도 복구가 가능한 실무 단골 출제 RAID 레벨은?
RAID 5
Q7. 성능을 높이는 스트라이핑(RAID 0)과 안정성을 높이는 미러링(RAID 1) 구조를 동시에 결합하여 사용하는 복합 RAID 형태는?
RAID 1+0 (또는 RAID 10)
Q8. Windows Server의 파일 시스템 중 데이터 압축, 파일 암호화(EFS), 개별 사용자 권한 설정 등을 지원하는 고성능 표준 파일 시스템은?
NTFS (New Technology File System)
Q9. 컴퓨터의 전원이 켜진 상태에서 장비의 손상이나 가동 중단 없이 하드디스크나 장치를 즉시 장착하거나 분리할 수 있는 기능을 뜻하는 용어는?
핫 플러깅 (Hot Plugging) 또는 핫 스왑 (Hot Swap)
Q10. Windows Server의 작업 관리자에서 실행 중인 프로세스의 강제 종료, CPU 및 메모리 점유율을 모니터링하기 위해 사용하는 단축키는?
Ctrl + Shift + Esc
Q11. Windows Server의 이벤트 뷰어에서 수집하는 로그 종류 3가지는 응용 프로그램 로그, 보안 로그, 그리고 나머지 하나는 무엇인가?
시스템 로그 (System Log)
Q12. Windows Server에서 하드디스크의 남은 공간을 사용자별로 제한하여 무분별한 저장소 독점을 막는 기능의 명칭은?
디스크 할당량 (Disk Quota)
Q13. Windows Server의 DNS 서비스에서 `도메인 영문 주소`를 기반으로 `숫자형 IP 주소`를 찾아내는 일반적인 조회를 무엇이라 하는가?
정방향 조회 (Forward Lookup)
Q14. 반대로 `숫자형 IP 주소`를 가지고 해당 컴퓨터의 `영문 도메인 네임`을 추적해 내는 DNS 조회 방식을 무엇이라 하는가?
역방향 조회 (Reverse Lookup)
Q15. Windows DNS 서버의 리소스 레코드 종류 중 호스트의 도메인 네임을 IPv4 주소와 1:1로 매핑해 주는 가장 기본 레코드는?
A 레코드 (Address 레코드)
Q16. Windows DNS 서버 리소스 레코드 중 도메인의 별칭(Alias)을 지정할 때 사용하여 하나의 IP에 여러 도메인 이름을 연결할 수 있게 돕는 레코드는?
CNAME (Canonical Name) 레코드
Q17. Windows DNS 서버 리소스 레코드 중 역방향 조회(IP 주소를 도메인으로 변환)를 위해 사용되는 레코드 명칭은?
PTR (Pointer) 레코드
Q18. Windows DNS 서버 리소스 레코드 중 해당 도메인의 이메일 라우팅(메일 서버)을 지정하기 위해 사용하는 레코드는?
MX (Mail Exchanger) 레코드
Q19. Windows Server에서 프린터를 공유할 때 여러 대의 동종 프린터를 하나의 논리적인 프린터 드라이버로 묶어 출력 속도를 분산 처리하는 기술은?
프린터 풀링 (Printer Pooling)
Q20. Windows Server 성능 모니터에서 시스템 리소스 중 데이터를 저장하거나 불러올 때 병목 현상이 생기는지 진단하기 위해 확인하는 핵심 개체는?
PhysicalDisk (물리적 디스크)
Q21. 리눅스 시스템에서 가장 강력한 권한을 가진 최고 관리자 계정의 이름은 무엇인가?
root
Q22. 리눅스에서 파일이나 폴더의 읽기(r), 쓰기(w), 실행(x) 권한(Permission)을 변경할 때 사용하는 필수 명령어는?
chmod
Q23. 리눅스에서 특정 파일이나 디렉토리의 소유자(Owner) 및 소유 그룹을 변경할 때 사용하는 명령어는?
chown
Q24. 리눅스 터미널 창에서 "내가 지금 어떤 폴더 경로 안에 들어와 있는지" 현재 작업 디렉토리의 절대 경로를 보여주는 명령어는?
pwd (Print Working Directory)
Q25. 리눅스에서 현재 디렉토리 내에 존재하는 파일과 폴더의 목록을 보여주는 명령어는?
ls
Q26. 리눅스에서 텍스트 파일의 내용을 화면에 그대로 출력하거나, 여러 파일을 하나로 결합할 때 사용하는 명령어는?
cat (Concatenate)
Q27. 리눅스에서 파일의 내용을 페이지 단위로 화면에 나누어 출력해 주며, 스페이스바를 눌러 다음 페이지로 넘길 수 있는 명령어는?
more (또는 less)
Q28. 리눅스에서 빈 파일을 생성하거나, 이미 존재하는 파일의 수정 시간을 현재 시간으로 갱신해 주는 명령어는?
touch
Q29. 리눅스에서 새로운 디렉토리(폴더)를 생성할 때 사용하는 명령어는?
mkdir (Make Directory)
Q30. 리눅스에서 파일이나 디렉토리를 완전히 삭제할 때 사용하는 명령어는?
rm (Remove)
Q31. 리눅스에서 하드디스크의 전체 용량과 현재 사용 중인 디스크 공간의 정보를 사람이 보기 편하게(KB, MB 단위 등) 출력해 주는 명령어는?
df (Disk Free)
Q32. 리눅스 시스템의 CPU 및 메모리 사용량, 실행 중인 프로세스 목록을 실시간으로 화면에 갱신해 보여주는 모니터링 명령어는?
top
Q33. 리눅스에서 현재 메모리의 총용량, 사용 중인 용량, 비어있는 남은 용량 및 스왑(Swap) 메모리 상태를 파악하는 명령어는?
free
Q34. 리눅스 환경에서 새 사용자 계정을 추가 생성할 때 사용하는 행정 명령어는?
useradd (또는 adduser)
Q35. 사용자의 비밀번호를 설정하거나 변경할 때 사용하는 명령어는?
passwd
Q36. 어떤 파일 내부나 명령어 출력 결과 중에서 내가 원하는 특정 문자열(패턴)이 포함된 라인만 필터링해 찾아주는 검색 명령어는?
grep
Q37. 리눅스에서 현재 시스템에 작동 중인 모든 프로세스의 상태 및 PID(프로세스 아이디) 번호를 일괄 출력하는 명령어는?
ps (특히 ps -ef 또는 ps aux 형태로 많이 쓰임)
Q38. 비정상적으로 동작하거나 멈춰버린 특정 프로세스를 PID 번호 등을 이용해 강제 종료시키는 제어 명령어는?
kill
Q39. 리눅스 텍스트 터미널 환경에서 가장 널리 쓰이는 표준 화면 편집기(텍스트 에디터) 명령어는?
vi (또는 vim)
Q40. 시스템을 즉시 안전하게 종료하거나 리부팅할 때 사용하는 관리자용 명령어는?
shutdown (또는 halt, reboot)
Q41. 리눅스 파일 권한 기호 중 `rwxr-xr-x` 가 의미하는 8진수 파일 권한 숫자는 무엇인가?
755 (rwx=7, r-x=5, r-x=5)
Q42. 리눅스 파일 권한 중 소유자, 그룹, 타인 모두에게 읽기 및 쓰기 권한만 부여하고 실행 권한은 빼고 싶을 때 지정하는 8진수 숫자는?
666 (rw-=6, rw-=6, rw-=6)
Q43. 리눅스 시스템에서 사용자 계정의 이름, UID, 기본 홈 디렉토리, 기본 셸 정보가 줄줄이 보관된 핵심 설정 파일의 경로는?
/etc/passwd
Q44. 리눅스 사용자 계정들의 실제 비밀번호가 안전하게 일방향 암호화(해시)되어 저장되어 있는 보안 파일의 경로는?
/etc/shadow
Q45. 리눅스 부팅 시 자동으로 마운트(연결)할 하드디스크 파티션 및 저장장치의 목록 정보를 담고 있는 설정 파일은?
/etc/fstab
Q46. 리눅스에서 주기적이고 반복적인 예약 작업(스케줄러 프로그램)을 등록하고 자동 관리하기 위해 사용하는 데몬 및 설정 명령어 명칭은?
cron (또는 crontab)
Q47. 리눅스 환경에서 도메인 네임 서버(DNS)의 IP 주소를 수동으로 지정하고 등록해 두는 네트워크 설정 파일의 경로는?
/etc/resolv.conf
Q48. 리눅스의 뼈대가 되는 핵심 커널이 부팅될 때 필요한 초기화 램 디스크 이미지 파일들이 주로 모여 보관되는 디렉토리 경로는?
/boot
Q49. 리눅스에서 파일이나 디렉토리가 생성될 때 자동으로 차단할 권한 가리개를 세팅하여 기본 초기 권한 값을 결정해 주는 명령어(개념)는?
umask
Q50. 리눅스에서 파일이나 명령어를 찾기 위해 디렉토리 목록 데이터베이스를 실시간 고속 검색해 내는 명령어는?
locate (또는 find)
